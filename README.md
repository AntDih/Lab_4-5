**Технічна документація системи криптографічного захисту файлів на базі AES**


  **Механізми керування ключами та автентифікація паролів**
  
  Даний проєкт представляє собою програмну реалізацію системи захисту інформації, що забезпечує повний цикл безпечної обробки файлів. В основі рішення лежить поєднання сучасних криптографічних стандартів шифрування та механізмів гарантування цілісності даних. Для захисту паролів користувача застосовано алгоритм генерації похідних ключів PBKDF2-HMAC-SHA256. Високий рівень безпеки досягається завдяки використанню унікальної 16-байтної солі для кожної операції та встановленню ліміту у 310,000 ітерацій, що робить спроби перебору пароля ресурсомісткими та малоефективними для зловмисника.

  **Реалізація режимів шифрування та забезпечення цілісності**
  
  Ядром системи є два функціональні режими шифрування. Режим AES-GCM використовується як сучасний стандарт автентифікованого шифрування, що дозволяє одночасно захищати конфіденційність та перевіряти цілісність даних через вбудований тег. Альтернативний режим AES-CBC реалізовано за класичною схемою Encrypt-then-MAC, де після шифрування до блоку даних додається підпис HMAC-SHA256. Це гарантує, що будь-яка несанкціонована зміна зашифрованого вмісту буде миттєво виявлена системою до моменту фактичного дешифрування. Проєкт також включає систему метаданих у форматі JSON-заголовка, яка зберігає параметри шифрування безпосередньо у файлі, забезпечуючи автономність дешифрування.

  **Моніторинг, аудит безпеки та стійкість до втручань**
  
  Для моніторингу роботи системи та аудиту безпеки впроваджено механізм структурованого логування у форматі JSON. Журнал фіксує всі успішні операції, вимірює точний час виконання кожної дії в мілісекундах та детально реєструє спроби втручання в дані або помилки автентифікації. Практична стійкість системи до модифікацій підтверджується шляхом імітації пошкодження зашифрованих файлів, при яких криптографічне ядро блокує доступ до інформації, забезпечуючи високий рівень захисту від атак на цілісність. Проєкт орієнтований на застосування коректних криптографічних практик та може бути використаний у дослідницьких або освітніх цілях.
